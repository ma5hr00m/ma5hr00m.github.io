0x01 Fedora系统结构目录

ma5hr00m此时使用的是Fedora系统. 他准备先来了解一下基本的系统结构目录.

/bin

存放系统常用命令，用户可以在终端中运行的命令都在这个文件夹中。

/bin是Linux系统中最常用的目录之一，包含了系统中需要频繁使用的命令，这些命令在系统启动时就已经加载到内存中，可以被所有用户使用，/bin目录下都是二进制可执行文件。

/sbin

存放系统管理员使用的命令。

/sbin是Linux系统中用于存储系统管理员使用的可执行文件的目录。它通常包含管理网络，配置硬件和系统安全等方面的命令。与/bin目录不同，/sbin目录中的命令通常需要root权限才能运行。例如，/sbin目录可能包含ifconfig，route和iptables等命令。另外，在/sbin目录下的一些命令可能在系统启动时自动运行，用于配置网络接口，初始化文件系统等。

/bin里面的指令用于具体应用，而**/sbin**里面的指令用于系统管理。

/etc

存放系统配置文件。

/etc目录是linux系统根目录下的一个重要目录，它存储着系统的配置文件。在这个目录中，有很多文件和子目录，每个文件都用来配置不同的系统组件，存储着系统的大量配置信息，如果不慎修改了这些配置文件，可能会导致系统无法正常启动。

/lib

存放系统库文件。

/lib目录是linux系统根目录下的一个重要目录，它主要用于存放系统运行时需要的共享库文件。

库文件是计算机上的一类文件，提供给使用者一些开箱即用的变量、函数或类。

库文件分为静态库和动态库，静态库和动态库的区别体现在程序的链接阶段：静态库在程序的链接阶段被复制到了程序中；动态库在链接阶段没有被复制到程序中，而是程序在运行时由系统动态加载到内存中供程序调用。使用动态库系统只需载入一次，不同的程序可以得到内存中相同的动态库的副本，因此节省了很多内存，而且使用动态库也便于模块化更新程序。

在操作系统中，许多应用程序并不是只有一个完整的可执行文件，大多数程序模块被分割成一些相对独立的动态库。当我们执行某一个程序时，相应的动态库文件就会被调用。一个应用程序可使用多个动态库文件，一个动态库文件也可能被不同的应用程序使用，这样的动态库文件被称为共享库文件。

来自百度百科

该目录下存放的共享库文件一般以.so作为后缀，如libc.so、libm.so等。这些库文件是系统核心功能所需的，比如C语言标准库、数学库、文件系统库等。

除了存放共享库文件之外，/lib目录还可能存放一些其他文件，比如配置文件、脚本等。这些文件也是系统运行时需要的。

/dev

存放系统设备文件。

在Linux系统中，所有的设备都被视为文件，并且都存储在/dev目录中。这些设备文件包括硬盘、光驱、键盘、鼠标、网络接口、打印机等，通常是虚拟文件，由内核驱动程序提供支持，不占用实际空间。

这些虚拟文件通常分为两类：字符设备文件和块设备文件。字符设备文件通常用于读写数据，如键盘、串口等。块设备文件通常用于读写块数据，如硬盘、光驱等。

块数据通常指的是按照固定大小的块来存储数据，例如硬盘或CD/DVD等存储介质上的数据。这种方式在读取和写入数据时会更有效率，因为硬件设备可以一次读取或写入一个块的数据。

数据通常指的是没有固定大小的数据，例如文本文件或图像等。这种方式可能更灵活，但在读取和写入数据时可能会更慢，因为硬件设备必须读取或写入整个文件。

/dev目录中的文件通常由系统自动创建，但也可以使用mknod或MAKEDEV命令手动创建。

/proc

存放系统进程信息。

/proc是Linux内核中的一个虚拟文件系统，它在内存中存储着关于系统和进程的信息。它提供了一种以文件形式访问系统数据的方式。这些文件都是只读的，通常用于监控系统和进程状态。

/proc目录下包含了许多子目录和文件，每个都对应着一个不同的系统和进程信息。例如：

  • /proc/cpuinfo：包含了关于CPU的信息。
  • /proc/meminfo：包含了关于内存的信息。
  • /proc/mounts：包含了关于已经挂载的文件系统的信息。
  • /proc/[PID]：每个进程在/proc目录下都有一个对应的子目录，其中包含了关于进程的信息，如进程状态，内存使用等。
  • …

/var

存放系统运行过程中产生的文件，如日志文件。

/var目录是linux系统中用来存放可变数据的目录。它的内容是随着系统运行而变化的，比如日志文件、缓存文件、运行进程的PID文件等。常见的子目录有：

  • /var/log：系统日志文件目录。包含了各种系统日志文件，如系统日志、应用程序日志等。
  • /var/run：存放系统进程的PID文件目录。每个正在运行的进程都会在这个目录下创建一个以PID命名的文件，用来保存该进程的PID。
  • /var/lock：存放文件锁目录。当多个进程同时访问同一个文件时，会在这个目录下创建一个以文件名命名的文件锁，用来防止文件被多次写入。
  • /var/cache：缓存目录。包含了各种应用程序的缓存数据，如浏览器缓存、DNS缓存等。
  • /var/lib：存放各种应用程序的数据库和配置文件目录。如MySQL数据库、Apache配置文件等。

/tmp

临时文件夹。

/tmp目录中保存了linux操作系统和其他正在执行的应用层软件需要的一些临时文件。每次重启时，系统都会自动清理临时文件。

/home

用户的主目录。

/root

系统管理员的主目录。

/usr

存放系统程序和文档。

在Linux系统中，所有的用户帐户都会被存储在/user目录下。每个用户帐户都会有一个独立的目录，这些目录的名字就是用户帐户的用户名。

例如，当用户名为“john”时，该用户帐户会被存储在/user/john目录下。

在/user目录下还有一些其他目录，例如/user/bin和/user/sbin，这些目录用来存储用户可执行文件。

/opt

存放可选软件。

/opt目录是Linux系统根目录下的一个特殊目录，它是用来存放第三方软件的安装目录。opt是“optional”的缩写，意思是可选的。

该目录中的软件通常是独立的，不会与系统自带的软件产生冲突，因此可以在不影响系统正常运行的情况下随时卸载和重新安装。这些软件通常也不会被系统升级自动更新，因此需要用户自行维护和更新。这些软件通常都有自己的二进制可执行文件、配置文件、库文件和帮助文档，这些文件都会被放置在/opt目录下的各自的子目录中。

/mnt

挂载其他文件系统的目录。

/mnt目录是Linux系统中用来挂载其他文件系统（如磁盘分区、移动存储设备、网络共享等）的目录。

挂载是指将外部文件系统连接到Linux系统中，使得Linux系统可以访问外部文件系统中的文件和目录。

/mnt目录中的文件夹名称通常与其挂载的文件系统的名称相关，例如/mnt/sda1表示将sda1分区挂载到/mnt/sda1目录中。

/media

挂载外部设备的目录。

/media目录是linux系统中用来存放外接设备的目录。当我们将一个外接设备（如U盘、硬盘等）插入电脑时，系统会自动将其挂载到/media目录下。挂载的设备会以其名称命名的目录形式存在于/media目录下。

例如，我们插入一个名为“U盘”的U盘，系统会在/media目录下创建一个名为“U盘”的目录，我们就可以在这个目录中读取和写入U盘中的文件。

/media目录下还有一个很重要的子目录叫/media/cdrom，这是系统默认挂载光盘的目录。当我们插入一张光盘时，系统会自动将其挂载到/media/cdrom目录下，我们就可以在这个目录中读取和写入光盘中的文件。

/boot

存放引导文件。

/boot目录是Linux系统中很重要的一个目录，主要用来存放系统启动所需的文件。在这个目录下常见的文件包括：

  1. vmlinuz：这是Linux系统的内核文件，是系统启动时首先被加载的文件。
  2. initrd：这是预加载的内存映像文件，在启动过程中加载驱动程序和其他必要的文件。
  3. grub：这是系统启动管理器文件，用来管理系统启动过程中的各种设置。
  4. config：这是grub配置文件，用来配置系统启动过程中的各种参数。
  5. System.map：这是系统的内核映像文件，用来显示系统中的各种符号。
  6. 文件夹：grub文件夹存放grub的配置文件，EFI文件夹存放系统启动过程中需要的文件。

/boot目录下的文件是系统启动过程中非常重要的文件，如果这些文件出现问题，系统就无法启动。

/srv

存放服务的数据文件

/srv目录是一个linux系统中用来存放特定服务数据的目录，它是根据FHS（Filesystem Hierarchy Standard，文件系统层次标准）规范规定的。

FHS规定，/srv目录是用来存放特定服务的数据的目录。这些数据可能是静态的或动态的，可能是由系统管理员维护的或由第三方维护的。

在/srv目录中，可能会存放如下类型的数据：

  • Web服务器的网站文件，如HTML、CSS和JavaScript文件
  • FTP服务器的文件
  • 数据库服务器的数据文件
  • 邮件服务器的电子邮件存档文件
  • 云存储服务的文件

0x02 dnf包管理器

包

先了解一下什么是”包”.

linux系统中的 “包” 通常指得是一个应用程序, 可以是命令行工具, GUI应用程序或其他软件库. “ 包 “ 本质上是一个存档目录, 包含二进制可执行文件, 配置文件, 有时还包含依赖关系的信息.

包管理器

再来了解一下包管理器.

在旧时代, 软件是从源代码处安装, 用户需要根据 README 文件编译该软件, 同时处理软件所需要的依赖关系.

为了简化操作, Linux发行版创建了自己的打包格式, 为终端用户提供随时可用的二进制文件 ( 预编译软件 ) , 以便安装软件, 同时提供一些元数据 ( 版本号、描述 ) 和依赖关系.

Debian创建了deb打包系统, Red Hat则创建了rpm打包系统.

为了与打包系统进行交互, 用户就需要一个管理器.

包管理器并非Linux系统独有的概念, 而是一个通用的概念, 很多软件甚至编程语言都拥有自己的包管理器.

比如, 你会发现python语言有自己的包管理器: pip.

包管理器不一定基于命令行, 也存在图形化的包管理器.

包管理器的运作

绝大多数Linux发行版都会有自己的软件仓库 ( software repository ), 可以理解为软件包的集合, 一个软件仓库内部包含很多不同的软件包.

软件仓库内包含有元数据文件, 该文件包含了软件包的名称, 版本号等基本信息.

使用包管理器时, 包管理器会先与这些元数据进行交互. 包管理器现在本地创建一个元数据的缓存. 使用update指令时, 包管理器会通过引用软件仓库中的元数据来更新本地元数据缓存.

运行安装指令时, 包管理器就会引用这个缓存, 如果在其中可以找到所需包的信息, 它就会连接到对应的仓库, 将包下载到本地. 同时, 包管理器会自动处理该包的依赖关系, 将依赖包与所需包一同下载安装.

除了基本的安装操作, 用户还可以自定义包管理器的配置文件, 根据自己的需要对包管理器进行修改, 如防止某个特定的包自动更新等.

dnf包管理器

现在来康康dnf.

dnf最早出现于Fedora 18这个发行版中, 是英文词组 “ Dandified YUM “ 的缩写.

dnf的功能是用于安装、更新、卸载Linux系统中的软件, 自Fedora 22开始, dnf取代了yum, 正式成为Fedoro的包管理器.

作为新一代的软件包管理器, dnf包管理器克服了yum包管理器的一些瓶颈, 提升了包括用户体验, 内存占用, 依赖分析, 运行速度等多方面的内容.
